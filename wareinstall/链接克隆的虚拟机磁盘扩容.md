1. 核心选择

   立即分配所有磁盘空间 (A)： 不勾选。

原因： 勾选后，VMware 会立即在你的物理机（Windows）硬盘上占用 100GB。不勾选的话，它是“动态增加”的，你用多少，物理机才占用多少。

将虚拟磁盘拆分成多个文件 (M)： 保持默认勾选。

原因： 这是 VMware 默认推荐的。虽然性能比单文件略低一点点（几乎感觉不出来），但它让虚拟机的备份、拷贝和跨电脑移动变得非常方便。

2. 接下来的操作流程
   点击“下一步”，完成添加。

开机（启动你的 bjc55 虚拟机）。

合并空间（最关键一步）：

进入终端执行以下命令，把这 100G 给到根目录：

① 确认新磁盘识别
Bash
fdisk -l
你应该能看到一个 /dev/sdb（大小显示约 107GB）。

② 创建物理卷 (PV)
Bash
pvcreate /dev/sdb
③ 扩展卷组 (VG)
Bash
vgextend centos /dev/sdb
④ 扩展逻辑卷 (LV) 并 刷新文件系统
Bash
# 一键分配所有新空间
lvextend -l +100%FREE /dev/mapper/centos-root

# 刷新 XFS 文件系统（针对 CentOS 7）
xfs_growfs /dev/mapper/centos-root
3. 验证
   操作完成后，再输入：

Bash
df -h
你会看到根目录（/）的容量已经从 17G 瞬间变成了 117G 左右。

检查文件系统类型：

Bash
df -T /
如果 Type 列显示是 xfs，用上面的 xfs_growfs /。

如果 Type 列显示是 ext4，请改用这个命令：

Bash
resize2fs /dev/mapper/centos-root
